## Idea central

La mayoría de los estudios de software construyen herramientas. La mayoría de los estudios de arte crean experiencias. Jifrex hace algo diferente: **trata el software como material artístico y estructural** — igual que un arquitecto trata al concreto o un compositor trata al sonido.

> *"¿En qué serás el único?"* — Thiel

La respuesta: en la intersección de **rigor técnico de software (AWS, MCP, plugin dev) + lenguaje artístico computacional (TouchDesigner, MaxMSP, RNBO) + capacidad de investigación y laboratorio**. Eso no existe como oferta empaquetada en el mercado hispanohablante. Ese es el foso.

## BLOQUE 1 — Propuesta de valor

**Diseñamos y construimos software como estructura:** plugins, piezas interactivas, entornos sonoros y visuales en tiempo real, y arquitecturas cloud — desde una lógica artística e investigativa, no solo funcional.

### Tres capas de valor

**Capa 1 — Herramientas (Plugins / SaaS)**  
Software que otros creativos y técnicos usan. El valor es productividad + posibilidad creativa que antes no existía.
- Plugins para DAWs (Ableton, Max for Live, RNBO exportado a VST/web)
- Herramientas propias para flujos de trabajo en instalaciones interactivas
- SaaS para nichos creativos que el mercado masivo ignora

**Capa 2 — Proyectos (Comisiones e instalaciones)**  
Piezas interactivas únicas para clientes e instituciones.
- Instalaciones audiovisuales interactivas (TouchDesigner + sensores + audio reactivo)
- Entornos de performance en vivo (Ableton + MaxMSP + RNBO)
- Sistemas de sonido e imagen generativo para espacios

**Capa 3 — Infraestructura y conocimiento (AWS + MCP Labs)**  
Arquitectura cloud para proyectos creativos y culturales.
- Laboratorios MCP: servidores MCP para flujos de trabajo con IA
- Soluciones AWS para distribución de contenido, streaming, procesamiento de audio/video
- Investigación y documentación publicable (el estudio como laboratorio)

**Por qué importa la distinción artística:** el mercado técnico puro compite por precio y velocidad. El mercado artístico puro no tiene infraestructura. Jifrex vive en el medio — y eso permite cobrar por criterio, no solo por horas.

## BLOQUE 2 — Segmentos de mercado

**Segmento A — Artistas y músicos con necesidades técnicas avanzadas ⭐**  
Compositores, performers, artistas sonoros y visuales que necesitan herramientas que no existen en el mercado.  
Dolor: el software masivo (Ableton, Max) tiene límites; necesitan extensiones a medida.  
Disposición a pagar: alta si el plugin resuelve un problema concreto.  
Canal: comunidades de MaxMSP, Cycling '74, foros de Ableton.

**Segmento B — Festivales, museos e instituciones culturales ⭐**  
Espacios que quieren experiencias interactivas pero no tienen el equipo técnico.  
Dolor: dependen de estudios de producción que no entienden el lenguaje artístico.  
Disposición a pagar: alta, especialmente con fondos de cultura o patrocinios.  
Canal: redes de gestores culturales, convocatorias de arte y tecnología.

**Segmento C — Empresas tech / startups con necesidades creativas**  
Empresas que quieren experiencias de producto diferenciadas (onboarding interactivo, interfaces sonoras).  
Dolor: los estudios de UX no entienden audio/interacción en tiempo real.  
Disposición a pagar: muy alta en B2B.  
Canal: LinkedIn, demos técnicas, GitHub con proyectos open source.

**Segmento D — Desarrolladores y técnicos creativos (SaaS / comunidad)**  
Usuarios de TouchDesigner, MaxMSP, Ableton que compran plugins o toman cursos.  
Modelo: ingresos recurrentes de bajo ticket, alta escala.  
Canal: Gumroad, plugin marketplaces, YouTube, comunidades online.

## BLOQUE 5 — Fuentes de ingresos

| Fuente | Modelo | Horizonte | Potencial |
|---|---|---|---|
| Plugins (Max for Live, VST, web audio) | Venta única / suscripción | 0–6 meses | Medio — escalable |
| Proyectos de instalación / performance | Fee por proyecto | 0–6 meses | Alto por proyecto |
| MCP Server Labs (consultoría + build) | Proyecto + retainer | 3–9 meses | Muy alto (mercado emergente) |
| Cursos / talleres (TouchDesigner, MaxMSP) | Venta única / cohort | 3–9 meses | Medio — flujo constante |
| AWS solutions (diseño + despliegue) | Proyecto + comisión AWS | 3–9 meses | Alto B2B |
| SaaS creativo | Suscripción mensual | 6–18 meses | Alto — requiere desarrollo previo |
| Investigación / residencias artísticas | Becas + fondos | 6–12 meses | Variable |

**Orden Lean recomendado:**
1. Plugins + primer proyecto de instalación (prueba de demanda rápida, bajo costo)
2. MCP Labs (mercado caliente, diferenciador técnico claro)
3. SaaS cuando haya feedback real de usuarios de plugins

## BLOQUE 9 — Estructura de costes

**Costes fijos esenciales**
- Licencias: Ableton Suite, MaxMSP, TouchDesigner (comercial), AWS (pago por uso)
- Infraestructura de desarrollo: hardware, repositorios, herramientas CI/CD
- Tiempo de I+D — el mayor costo; hay que protegerlo

**Costes variables por proyecto**
- Hardware de instalación (alquilar hasta tener flujo estable)
- Colaboradores técnicos por proyecto
- Viajes para instalaciones presenciales

**Principio:** mantener costes fijos bajos. Los plugins y el SaaS generan margen incremental una vez desarrollados. Priorizar activos digitales sobre equipamiento físico.

## Ventaja estratégica: MCP Servers

Los laboratorios de MCP servers son el activo más estratégico en este momento:

- El mercado MCP es emergente — hay muy pocos actores con competencia técnica real
- La intersección MCP + arte computacional (TouchDesigner, Ableton) no existe como oferta
- AWS + MCP es una combinación que las empresas tech necesitan y no saben cómo construir
- El estudio puede posicionarse como referente antes de que el mercado se llene

**Acción concreta:** construir y publicar un MCP server propio relacionado con audio o visual generativo — demo pública que funcione como tarjeta de presentación técnica. El directorio `mcp-labs/` en RepoJifrex ya está listo para esto.
